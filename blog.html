Question/Answer :

1.What is the Document Object Model (DOM) ?

The Document Object Model (DOM) is a way for your browser to understand and control the content of a webpage. Think of it like a map or a guide to all the parts of a webpage, like headings, paragraphs, images, etc.

When a webpage loads, the browser looks at the HTML and builds a tree that shows how all the parts of the page are related. Each part of the page, like a title or a picture, is a "node" or "object" on this tree. You can change what’s on the page using JavaScript. Want to make a button do something when clicked? Want to change text or add new elements? The DOM lets you do all that!

2.How do you select an element from the DOM ?

To pick an element from a webpage using JavaScript, you have a few simple ways:

  • By ID: If the element has an ID (like a name tag), you can use getElementById(). IDs are unique, so this method picks only one element.
  • By Class: If the element has a class (like a label that can be shared), you can use getElementsByClassName(). This can pick more than one element because many elements can have the same class.
  • By Tag: If you want to select all elements of a certain type, like all div or p tags, you use getElementsByTagName(). This will pick all elements with that tag.
  • By CSS Selector: You can use querySelector() to pick the first element that matches a CSS rule, like an ID, class, or tag. If you want to select all matching elements, use querySelectorAll().

3.What is event delegation in the context of the DOM, and why it is useful ?

Event delegation is a way to handle events in the DOM by using a single event listener on a parent element, instead of adding multiple listeners to individual child elements.

Event delegation is a way to handle events in the DOM by using a single event listener on a parent element, instead of adding multiple listeners to individual child elements. Why is it useful? Fewer event listeners: Instead of adding a separate event listener to each child element, you just add one to the parent. This reduces memory usage and improves performance.
Handles future elements: If new child elements are added later, you don’t need to add new listeners. The parent can handle events for both current and future child elements.
Example : Imagine you have a list of items, and you want something to happen when you click any item. Instead of adding a click listener to each item, you can put the listener on the list itself (the parent). The parent will know which item was clicked by checking where the event happened. Event delegation makes your code simpler and faster, especially when dealing with many elements.

4.how do you manipulate an element's attributes and styles using DOM ?

To change an element’s attributes and styles in the DOM:

If you want to change - src, href, id: setAttribute(): This changes an attribute, like changing the source of an image or the link of a button.
Example: If you have an image, you can change its source like this:
imageElement.setAttribute("src", "newImage.jpg");
You can also change common attributes directly. Example: Change an element's id: element.id = "newId"; Changing Styles (like color, size, background): style.property: You can change how an element looks by using the style property. Example: Make text red: element.style.color = "red"; classList: You can add or remove classes that are defined in CSS. Example: Add a new class to the element: element.classList.add("newClass"); Change attributes with setAttribute() or directly (like element.id = "newId";). Change styles with style (like element.style.color = "blue";) or classList.
